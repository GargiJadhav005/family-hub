# 🚀 DEPLOYMENT & ACCOUNT SETUP GUIDE

## ⚠️ IMPORTANT: Before Deploying to Production

All demo accounts have been removed from this codebase. The system is now ready for real deployment with production accounts.

## 📋 SYSTEM OVERVIEW

**Available User Roles:**
- 👨‍🏫 **Teacher** - Manages students, homework, attendance, quizzes, meetings
- 🧑‍🎓 **Student** - Views homework, attempts quizzes, checks scores
- 👨‍👩‍👧 **Parent** - Monitors child's progress, views attendance and marks
- 🛠️ **Admin** - System administrator with full control

## 🔧 STEP 1: Environment Configuration

### Backend Setup (.env)

1. **Copy and configure the backend environment:**
   ```bash
   cd backend
   cp .env.example .env
   ```

2. **Update these critical values in `backend/.env`:**

   ```env
   # ✅ REQUIRED: Your MongoDB cluster details
   MONGODB_URI=mongodb+srv://your_username:password@cluster.mongodb.net/?appName=your_app
   MONGODB_DB_NAME=family_hub
   
   # ✅ REQUIRED: Generate a strong JWT secret
   # Run: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"
   JWT_SECRET=your_generated_64_character_hex_string_here
   
   # ✅ REQUIRED: Where your frontend is hosted
   FRONTEND_URL=http://your-domain.com:3000
   ```

### Frontend Setup (.env.local)

1. **Create frontend environment file:**
   ```bash
   cp .env.example .env.local
   ```

2. **Update the API URL:**
   ```env
   # Point to your backend server
   VITE_API_URL=http://your-backend-domain:5000/api
   ```

## 👤 STEP 2: Create Admin Account (First-Time Setup)

The system comes with one admin seeding script (`seedAdmin.ts`) that you should run ONCE to create the admin account.

### Option A: Using npm script (Recommended)

```bash
cd backend
npm run seed:admin
```

### Option B: Using tsx directly

```bash
cd backend
npx tsx src/scripts/seedAdmin.ts
```

### What happens:
- ✅ Creates admin user with email: `admin@google.com`
- ✅ Creates initial password: `Admin@123`
- ⚠️ **IMPORTANT**: Change password immediately after first login!

## 👨‍🎓 STEP 3: Create Real User Accounts (Ongoing)

### Admin Creates Users via Admin Dashboard

1. **Login as Admin**
   - Email: `admin@google.com`
   - Password: `Admin@123` (after you change it on first login)

2. **Go to Admin Dashboard → User Management**

3. **Create users:**
   - Click "Add Teacher" button
   - Enter: Name, Email, Password, Department, Class
   - System generates API token automatically
   - Share credentials with the teacher securely

### Teachers Create Students via Enroll Page

1. **Login as Teacher**
   - Use credentials created by admin

2. **Go to Teacher → Enroll Students**

3. **Create students:**
   - Enter: Student Name, Parent Name, Class
   - System generates unique login credentials automatically
   - Parent and Student logins are auto-created
   - Share credentials securely with parents

### Parents & Students Login

Parents and students use credentials generated by teachers:
- **Student**: `student_email@domain.com` / auto-generated password
- **Parent**: `parent_email@domain.com` / auto-generated password

## 📦 STEP 4: Build & Deploy

### Local Development

```bash
# Terminal 1: Start Backend
cd backend
npm run dev
# Runs on http://localhost:5000

# Terminal 2: Start Frontend
npm run dev
# Runs on http://localhost:5173
```

### Production Build

```bash
# Build backend
cd backend
npm run build
npm start

# Build frontend
npm run build
npm run preview
```

### Docker Deployment (Optional)

Create `Dockerfile` for backend:
```dockerfile
FROM node:18-alpine
WORKDIR /app
COPY backend/package*.json ./
RUN npm install
COPY backend/src ./src
RUN npm run build
CMD ["npm", "start"]
EXPOSE 5000
```

## 🆘 TROUBLESHOOTING

### Issue: "Invalid credentials" on login
**Solution:** 
- Verify user exists in database
- Check email and password are correct
- Verify JWT_SECRET is set in backend/.env

### Issue: Frontend can't connect to backend
**Solution:**
- Check VITE_API_URL in .env.local matches backend URL
- Verify backend is running on correct port (5000)
- Check CORS is configured (FRONTEND_URL in backend/.env)
- Check for network/firewall issues

### Issue: Admin seeding fails
**Solution:**
- Verify MONGODB_URI is correct and database is accessible
- Check admin user doesn't already exist
- Delete the user if it exists, then run seed again

### Issue: Password hashing errors
**Solution:**
- Install missing dependencies: `npm install`
- Check bcryptjs is in package.json
- Restart backend server
